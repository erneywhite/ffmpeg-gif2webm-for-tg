# gif2tgwebm

Простой bash-скрипт для пакетной конвертации GIF в Telegram-совместимые видео‑стикеры и эмодзи (формат WEBM + VP9) с автоматическим подбором качества под лимит размера.

- Поддерживает два режима: `sticker` и `emoji`
- Соблюдает ограничения Telegram по длительности, FPS, размеру кадра и весу файла
- Автоматически повышает CRF, пока результат не уложится в целевой размер

---

## Требования

- `bash`
- `ffmpeg` с поддержкой `libvpx-vp9`
- Одна из сред:
  - macOS (используется `stat -f%z`)
  - Windows с WSL (внутри WSL Linux-дистрибутив, используется `stat -c%s` при адаптации скрипта)
  - Linux (аналогично WSL: `stat -c%s`)

> В текущем виде скрипт использует вариант `stat -f%z`, который работает на macOS. Для Linux/WSL см. раздел «Кроссплатформенность».

---

## Ограничения Telegram

Скрипт заточен под официальные требования для видео‑стикеров и эмодзи:

**Общее:**

- Формат: `.webm`
- Кодек: VP9
- Без аудио
- FPS: до 30
- Длительность: до 3 секунд

**Стикеры:**

- Одна сторона ровно 512 пикселей, другая ≤ 512
- Размер файла до 256 KB

**Эмодзи:**

- Разрешение 100×100 пикселей
- Размер файла до 64 KB

---

## Как работает скрипт

- Берёт все `*.gif` из входной директории (`SRC_DIR`, по умолчанию `gif/`)
- Для каждого GIF:
  - масштабирует под нужный размер:
    - 512×512 (c сохранением пропорций) для стикеров
    - 100×100 для эмодзи
  - кодирует в WEBM (VP9, `yuva420p`)
  - стартует с базового значения CRF
  - проверяет размер файла
  - если размер больше лимита:
    - увеличивает CRF на шаг
    - перекодирует, пока не уложится в лимит или не достигнет максимального CRF
- Результат сохраняет в выходную директорию (`OUT_DIR`, по умолчанию `webm/`)

Исходные GIF *не изменяются*.

## Установка и подготовка

### Вариант: macOS

1. Установите ffmpeg (через Homebrew):

```bash
brew install ffmpeg
```

2. Склонируйте репозиторий:

```bash
git clone https://github.com/yourname/gif2tgwebm.git
cd gif2tgwebm
```

3. Сделайте скрипт исполняемым:

```bash
chmod +x gif2tgwebm.sh
```

4. Подготовьте директории:

```bash
mkdir -p gif webm
# Скопируйте свои .gif в папку gif/
```

### Вариант: Windows через WSL + PowerShell

Скрипт bash‑овый, поэтому в Windows удобнее всего запускать его через WSL.

#### 1. Установка WSL и Linux-дистрибутива

Откройте PowerShell **от имени администратора** и выполните:

```powershell
wsl --install
```

- По умолчанию установится Ubuntu
- После перезагрузки настройте пользователя в WSL

Запустить оболочку WSL:

```powershell
wsl
```

#### 2. Установка ffmpeg внутри WSL

В терминале WSL (например, Ubuntu):

```bash
sudo apt update
sudo apt install ffmpeg -y
ffmpeg -version
```

#### 3. Клонирование репозитория и подготовка

Внутри WSL:

```bash
git clone https://github.com/yourname/gif2tgwebm.git
cd gif2tgwebm
chmod +x gif2tgwebm.sh

mkdir -p gif webm
# Скопируйте свои .gif в папку gif/
```

> Можно хранить репозиторий в домашней директории WSL, например `/home/username/gif2tgwebm`.

#### 4. Запуск скрипта внутри WSL

```bash
./gif2tgwebm.sh           # режим sticker по умолчанию
./gif2tgwebm.sh emoji     # режим emoji
./gif2tgwebm.sh sticker 28
./gif2tgwebm.sh emoji 40
```

#### 5. Запуск скрипта из PowerShell через WSL

Чтобы не заходить вручную в WSL, можно запускать скрипт напрямую из PowerShell, если путь совпадает и для Windows, и для WSL:

```powershell
cd path\to\gif2tgwebm

wsl ./gif2tgwebm.sh
wsl ./gif2tgwebm.sh emoji
wsl ./gif2tgwebm.sh sticker 28
wsl ./gif2tgwebm.sh emoji 40
```

Условия:

- В WSL по этому пути должен лежать репозиторий со скриптом и папками `gif` / `webm`
- ffmpeg установлен **внутри** WSL
- Скрипт отмечен как исполняемый (`chmod +x gif2tgwebm.sh` в WSL)

## Использование

### Режим стикеров (по умолчанию)

Конвертация всех GIF в `gif/` в Telegram‑стикеры (512 px, лимит 256 KB):

```bash
./gif2tgwebm.sh
# или явно:
./gif2tgwebm.sh sticker
```

С указанием стартового CRF (ниже CRF = лучше качество, больше размер):

```bash
./gif2tgwebm.sh sticker 28
```

### Режим эмодзи

Конвертация всех GIF в `gif/` в видео‑эмодзи (100×100, лимит 64 KB):

```bash
./gif2tgwebm.sh emoji
```

Более агрессивное сжатие (выше стартовый CRF):

```bash
./gif2tgwebm.sh emoji 40
```

Готовые WEBM‑файлы появляются в директории `webm/` с теми же именами, что и у GIF, но с расширением `.webm`.

## Переменные и настройки

Вверху скрипта находятся основные настройки по умолчанию:

```bash
SRC_DIR="gif"        # входные gif
OUT_DIR="webm"       # выходные webm

DEFAULT_MODE="sticker"   # MODE по умолчанию: sticker или emoji

MAX_DURATION="2.99"      # максимальная длительность (сек)
FPS="30"                 # FPS (до 30)

# Лимиты размера
TARGET_SIZE_STICKER=$((256 * 1024))  # 256 KB
TARGET_SIZE_EMOJI=$((64 * 1024))     # 64 KB

# Стартовые CRF по умолчанию
DEFAULT_CRF_START_STICKER=32
DEFAULT_CRF_START_EMOJI=36

CRF_STEP=2    # шаг увеличения CRF при переборе
CRF_MAX=46    # максимальный CRF
```

### Режимы и размеры

Режим задаётся аргументом `MODE`:

**sticker**

- масштаб: `scale=512:512:force_original_aspect_ratio=decrease`
- лимит размера: 256 KB (`TARGET_SIZE_STICKER`)
- стартовый CRF по умолчанию: `DEFAULT_CRF_START_STICKER`

**emoji**

- масштаб: `scale=100:100`
- лимит размера: 64 KB (`TARGET_SIZE_EMOJI`)
- стартовый CRF по умолчанию: `DEFAULT_CRF_START_EMOJI`

Запуск:

```bash
./gif2tgwebm.sh <MODE> <CRF_START>
```

Оба параметра необязательны:

- если `MODE` не указан → используется `DEFAULT_MODE`
- если `CRF_START` не указан → берётся соответствующий `DEFAULT_CRF_START_*` для выбранного режима

## Логика подбора качества (CRF)

Для каждого входного GIF:

1. Стартуем с `CRF_START`
2. Кодируем во временный файл `*_crfXX.webm`
3. Проверяем размер:
   - если `size <= TARGET_SIZE`:
     - переименовываем временный файл в итоговый
   - если `size > TARGET_SIZE`:
     - увеличиваем CRF на `CRF_STEP`
     - повторяем кодирование
4. Если CRF превысил `CRF_MAX`:
   - файл считается «неуложившимся в лимит»
   - временный файл удаляется
   - итоговый `.webm` для этого GIF не создаётся

## Кроссплатформенность

Сейчас в скрипте используется macOS‑вариант `stat`:

```bash
size=$(stat -f%z "$tmp_out")
```

Для Linux/WSL нужно заменить эту строку на:

```bash
size=$(stat -c%s "$tmp_out")
```

Если планируете использовать скрипт и на macOS, и в WSL/Linux, можно сделать простую проверку `uname` в скрипте и выбирать нужный вариант `stat` в зависимости от системы.

---

## Предупреждения и нюансы

- Скрипт пропускает файлы, для которых итоговый `.webm` уже существует в `OUT_DIR`.
- GIF лучше подавать уже с прозрачностью (если она нужна). Скрипт не «вырезает» фон и не делает цвет прозрачным – он только сохраняет альфу, присутствующую в исходнике.
- При очень сложной анимации и жёстких лимитах по размеру (особенно для эмодзи 64 KB) возможна заметная деградация качества на высоких значениях CRF.
