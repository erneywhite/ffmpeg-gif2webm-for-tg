# gif2tgwebm

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Shell: Bash](https://img.shields.io/badge/Shell-Bash-4EAA25?logo=gnubash&logoColor=white)](gif2tgwebm.sh)
[![FFmpeg](https://img.shields.io/badge/Requires-FFmpeg%20%2B%20libvpx--vp9-007808?logo=ffmpeg&logoColor=white)](https://ffmpeg.org)

Простой bash-скрипт для **пакетной конвертации GIF** в Telegram-совместимые видео‑стикеры и эмодзи (формат WEBM + VP9) с автоматическим подбором качества под лимит размера.

- Поддерживает два режима: `sticker` и `emoji`
- Соблюдает ограничения Telegram по длительности, FPS, размеру кадра и весу файла
- Автоматически повышает CRF, пока результат не уложится в целевой размер
- Не изменяет исходные GIF-файлы

## Требования

- `bash`
- `ffmpeg` с поддержкой кодека `libvpx-vp9`
- Одна из сред:
  - **macOS** — используется `stat -f%z` (текущий вариант по умолчанию)
  - **Linux** / **Windows WSL** — используется `stat -c%s` (см. раздел [Кроссплатформенность](#кроссплатформенность))

---

## Ограничения Telegram

Скрипт заточен под официальные требования для видео‑стикеров и эмодзи:

**Общее:**

- Формат: `.webm`
- Кодек: VP9
- Без аудио
- FPS: до 30
- Длительность: до 3 секунд

**Стикеры:**

- Одна сторона ровно 512 пикселей, другая ≤ 512
- Размер файла до 256 KB

**Эмодзи:**

- Разрешение 100×100 пикселей
- Размер файла до 64 KB

---

## Как работает скрипт

- Берёт все `*.gif` из входной директории (`SRC_DIR`, по умолчанию `gif/`)
- Для каждого GIF:
  - масштабирует под нужный размер (512×512 для стикеров, 100×100 для эмодзи)
  - кодирует в WEBM (VP9, `yuva420p`)
  - стартует с базового значения CRF
  - проверяет размер файла
  - если размер больше лимита — увеличивает CRF на шаг и перекодирует
- Результат сохраняет в `OUT_DIR` (по умолчанию `webm/`)

Исходные GIF *не изменяются*.

## Установка и подготовка

### macOS

1. Установите ffmpeg через Homebrew:

```bash
brew install ffmpeg
```

2. Склонируйте репозиторий:

```bash
git clone https://github.com/erneywhite/ffmpeg-gif2webm-for-tg.git
cd ffmpeg-gif2webm-for-tg
```

3. Сделайте скрипт исполняемым:

```bash
chmod +x gif2tgwebm.sh
```

4. Подготовьте директории и добавьте GIF:

```bash
mkdir -p gif webm
# Скопируйте свои .gif в папку gif/
```

### Linux

```bash
sudo apt update && sudo apt install ffmpeg -y
git clone https://github.com/erneywhite/ffmpeg-gif2webm-for-tg.git
cd ffmpeg-gif2webm-for-tg
chmod +x gif2tgwebm.sh
mkdir -p gif webm
```

> ⚠️ На Linux нужно изменить строку `stat` — см. раздел [Кроссплатформенность](#кроссплатформенность).

### Windows через WSL

Скрипт bash‑овый, поэтому в Windows удобнее всего запускать его через WSL.

#### 1. Установка WSL и Linux-дистрибутива

Откройте PowerShell **от имени администратора** и выполните:

```powershell
wsl --install
```

После перезагрузки настройте пользователя в WSL и запустите оболочку:

```powershell
wsl
```

#### 2. Установка ffmpeg внутри WSL

```bash
sudo apt update
sudo apt install ffmpeg -y
ffmpeg -version
```

#### 3. Клонирование репозитория

```bash
git clone https://github.com/erneywhite/ffmpeg-gif2webm-for-tg.git
cd ffmpeg-gif2webm-for-tg
chmod +x gif2tgwebm.sh
mkdir -p gif webm
```

> ⚠️ На WSL нужно изменить строку `stat` — см. раздел [Кроссплатформенность](#кроссплатформенность).

#### 4. Запуск из PowerShell напрямую

```powershell
cd path\to\ffmpeg-gif2webm-for-tg

wsl ./gif2tgwebm.sh
wsl ./gif2tgwebm.sh emoji
wsl ./gif2tgwebm.sh sticker 28
wsl ./gif2tgwebm.sh emoji 40
```

> Условия: ffmpeg установлен **внутри** WSL, скрипт отмечен как исполняемый.

## Использование

### Режим стикеров (по умолчанию)

Конвертация всех GIF в `gif/` в Telegram‑стикеры (512 px, лимит 256 KB):

```bash
./gif2tgwebm.sh
# или явно:
./gif2tgwebm.sh sticker
```

С указанием стартового CRF (ниже CRF = лучше качество, но больше размер):

```bash
./gif2tgwebm.sh sticker 28
```

### Режим эмодзи

Конвертация всех GIF в видео‑эмодзи (100×100, лимит 64 KB):

```bash
./gif2tgwebm.sh emoji
```

С более агрессивным сжатием:

```bash
./gif2tgwebm.sh emoji 40
```

Готовые WEBM‑файлы появляются в директории `webm/` с теми же именами, что и у GIF, но с расширением `.webm`.

### Пример вывода

```
MODE=sticker, CRF_START=32, TARGET_SIZE=262144B
Конвертирую: gif/cat.gif -> webm/cat.webm (режим: sticker)
  >262144B (310500 байт при CRF=32), пробую CRF=34
  OK: webm/cat.webm (241680 байт, CRF=34)
Готово.
```

## Переменные и настройки

Вверху скрипта находятся основные настройки по умолчанию:

```bash
SRC_DIR="gif"        # входные gif
OUT_DIR="webm"       # выходные webm

DEFAULT_MODE="sticker"   # MODE по умолчанию: sticker или emoji

MAX_DURATION="2.99"      # максимальная длительность (сек)
FPS="30"                 # FPS (до 30)

# Лимиты размера
TARGET_SIZE_STICKER=$((256 * 1024))  # 256 KB
TARGET_SIZE_EMOJI=$((64 * 1024))     # 64 KB

# Стартовые CRF по умолчанию
DEFAULT_CRF_START_STICKER=32
DEFAULT_CRF_START_EMOJI=36

CRF_STEP=2    # шаг увеличения CRF при переборе
CRF_MAX=46    # максимальный CRF
```

### Режимы и параметры

| Параметр         | `sticker`                                          | `emoji`              |
|------------------|----------------------------------------------------|----------------------|
| Масштаб          | `scale=512:512:force_original_aspect_ratio=decrease` | `scale=100:100`      |
| Лимит размера    | 256 KB                                             | 64 KB                |
| Стартовый CRF    | 32                                                 | 36                   |

Синтаксис запуска:

```bash
./gif2tgwebm.sh [MODE] [CRF_START]
```

Оба параметра необязательны: если `MODE` не указан — используется `DEFAULT_MODE`; если `CRF_START` не указан — берётся соответствующий `DEFAULT_CRF_START_*`.

## Логика подбора качества (CRF)

Для каждого входного GIF:

1. Стартуем с `CRF_START`
2. Кодируем во временный файл `*_crfXX.webm`
3. Проверяем размер:
   - если `size <= TARGET_SIZE` → переименовываем во итоговый файл ✅
   - если `size > TARGET_SIZE` → увеличиваем CRF на `CRF_STEP` и повторяем
4. Если CRF превысил `CRF_MAX`:
   - временный файл удаляется
   - итоговый `.webm` для этого GIF **не создаётся** ❌

## Кроссплатформенность

В скрипте по умолчанию используется macOS‑вариант `stat`:

```bash
size=$(stat -f%z "$tmp_out")
```

Для **Linux / WSL** замените эту строку на:

```bash
size=$(stat -c%s "$tmp_out")
```

Если нужна поддержка обеих платформ одновременно, добавьте автоопределение системы:

```bash
if [[ "$(uname)" == "Darwin" ]]; then
  size=$(stat -f%z "$tmp_out")
else
  size=$(stat -c%s "$tmp_out")
fi
```

## Предупреждения и нюансы

- Скрипт **пропускает** файлы, для которых итоговый `.webm` уже существует в `OUT_DIR`.
- GIF лучше подавать уже с прозрачностью (если она нужна). Скрипт не вырезает фон и не делает цвет прозрачным — он только сохраняет альфу, присутствующую в исходнике.
- При очень сложной анимации и жёстких лимитах по размеру (особенно для эмодзи 64 KB) возможна заметная деградация качества на высоких значениях CRF.
- Если файл не уложился в лимит даже при `CRF_MAX=46` — попробуйте вручную уменьшить `MAX_DURATION` или упростить анимацию исходного GIF.
